# 贡献指南

感谢您对 LilysToolKits 项目的关注！我们欢迎所有形式的贡献。

## 如何贡献

### 报告问题
- 在提交新问题之前，请先搜索现有的 issues
- 使用清晰、描述性的标题
- 提供详细的问题描述和复现步骤
- 包含您的环境信息（浏览器、操作系统等）

### 提交代码
1. **Fork 本仓库**
   ```bash
   # 在 GitHub 上点击 Fork 按钮
   ```

2. **克隆您的 Fork**
   ```bash
   git clone https://github.com/YOUR_USERNAME/lilystoolkits-h5.git
   cd lilystoolkits-h5
   ```

3. **创建功能分支**
   ```bash
   git checkout -b feature/amazing-feature
   ```

4. **进行更改**
   - 遵循现有的代码风格
   - 添加必要的注释
   - 确保代码在不同浏览器中的兼容性
   - 测试您的更改

5. **提交更改**
   ```bash
   git add .
   git commit -m "feat: 添加某个新功能"
   ```

6. **推送到您的 Fork**
   ```bash
   git push origin feature/amazing-feature
   ```

7. **创建 Pull Request**
   - 在 GitHub 上访问您的 Fork
   - 点击 "New Pull Request"
   - 填写详细的 PR 描述

## 代码规范

### HTML
- 使用语义化 HTML5 标签
- 保持代码缩进一致（2 个空格）
- 添加必要的注释

### CSS
- 使用 BEM 命名约定（如适用）
- 保持选择器简洁
- 添加响应式设计考虑

### JavaScript
- 使用 ES6+ 语法
- 函数和变量命名要有意义
- 添加 JSDoc 注释（如需要）
- 避免使用全局变量

### 提交信息格式
使用约定式提交格式：
- `feat:` 新功能
- `fix:` 修复 bug
- `docs:` 文档更新
- `style:` 代码格式化
- `refactor:` 代码重构
- `test:` 测试相关
- `chore:` 构建过程或辅助工具的变动

## 测试指南

- 在不同浏览器中测试您的更改（Chrome、Firefox、Safari、Edge）
- 测试移动设备上的响应式表现
- 确保现有功能没有被破坏

## 发布流程

- 项目维护者会定期合并 PR 并发布新版本
- 版本号遵循语义化版本规范
- 重要更改会在 release notes 中记录

## 获取帮助

如果您有任何问题或需要帮助：
- 在 GitHub Issues 中提问
- 查看项目文档
- 联系项目维护者
